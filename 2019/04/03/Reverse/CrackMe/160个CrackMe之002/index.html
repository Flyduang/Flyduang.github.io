160个CrackMe之002

发表于 2019-04-03 | 分类于 逆向分析 ， CrackMe

160个新手练习CrackMe，第二个。

0x1 环境

  • 系统：Windows XP SP3
  • 软件：Afkayas.1.exe
  • 工具：PEiD v0.95、吾爱破解专用版OllyDbg

0x2 分析

运行程序，需要输入正确的用户名和密码。

PEiD查壳，无壳，Microsoft Visual Basic 编写。

输入任意用户名和注册码，得到失败提示。

载入OD，搜索字符串，确定错误提示位置。

逆向分析数据流，找到关键跳转。

得益于OD的强大，继续向上发现比较函数。

在此处下断，F9运行到这里，从栈中得到正确的验证码和输入的假码。

上方挨着字符串拼接函数，将“AKA-”与正确的序列号连接起来。

继续分析，找到序列号生成算法。

生成流程为：

  1. 计算用户名长度。
  2. 长度与0x17CFB相乘。
  3. 第二步结果与用户名第一个字符的ASCII码相加。
  4. 将第三步结果转为十进制表示。
  5. 将十进制表示转为字符串类型。

例如用户名为test，注册码为AKA-390240，输入程序，注册成功。

分析完毕！
